# Agent Instructions — Credit Risk AI Agent

AI-powered credit risk analysis system. Predicts client default probability and explains reasoning via a ReAct agent backed by ML models and RAG. See [README.md](README.md) for project overview.

## Quick Start

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"

# Required — create before running the agent or server
echo "OPENROUTER_API_KEY=sk-or-..." > .env
```

```bash

pytest tests/ -x --cov --cov-report=xml
```

## Architecture

```
POST /chat
  └─ InputGuardrail.validate()          # block injections, off-topic, >4096 chars
      └─ react_agent.chat()             # LangGraph ReAct, GPT-4o-mini via OpenRouter
          ├─ predict_risk(json_str)     # loads joblib model → predict_proba
          ├─ query_data(question)       # stats queries on processed DataFrame
          ├─ explain_decision(json_str) # coefficient-based feature importance
          └─ search_docs(query)         # FAISS + sentence-transformers RAG
      └─ OutputGuardrail.sanitize()     # redact CPF, phone, email (LGPD)
```

Key files:
- [`src/agent/react_agent.py`](src/agent/react_agent.py) — `chat(query)` creates a fresh stateless agent per request
- [`src/agent/tools.py`](src/agent/tools.py) — 4 LangChain tools; models loaded once as module-level globals at import time
- [`src/agent/rag_pipeline.py`](src/agent/rag_pipeline.py) — FAISS index auto-built from `data/docs/*.txt` on first run; stored at `models/faiss_index.bin`
- [`src/serving/app.py`](src/serving/app.py) — FastAPI: `POST /chat`, `GET /health`, `GET /metrics`
- [`src/security/guardrails.py`](src/security/guardrails.py) — input validation + PII redaction

## Conventions

- **Module-level globals in tools.py**: `_model`, `_scaler`, `_df`, `_retriever` are loaded once at import. Do not add lazy loading — it breaks the existing pattern.
- **Missing feature values in `predict_risk`**: filled with dataset median from `_df[feat].median()`.
- **Agent is stateless**: `create_agent()` is called fresh for every `/chat` request — there is no conversation history across calls.
- **Hardcoded paths**: model and data paths are relative to the working directory (project root). Always run commands from the project root.
- **Function naming**: `verb_noun` (e.g., `remove_invalid_ages`, `predict_risk`, `explain_decision`).
- **Private globals**: leading underscore (`_model`, `_df`).

## Security & Compliance

- See [docs/OWASP_MAPPING.md](docs/OWASP_MAPPING.md) for the 6 threats addressed.
- See [docs/LGPD_PLAN.md](docs/LGPD_PLAN.md) for Brazil data privacy compliance plan.
- See [docs/RED_TEAM_REPORT.md](docs/RED_TEAM_REPORT.md) for 6 adversarial scenarios tested.
- All user input passes through `InputGuardrail` before reaching the agent. All output passes through `OutputGuardrail` before returning to the user.
- **Do not disable guardrails** in production code. Tests that bypass them should use explicit mocking, not removal.
- PII patterns to redact (Brazilian): CPF (`\d{3}\.?\d{3}\.?\d{3}-?\d{2}`), phone, email.
